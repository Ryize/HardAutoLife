# Модуль обработки подготовленных данных о движении транспорта


## Возможности

-  Загрузка и валидация данных в формате JSON
-  Восстановление траекторий по последовательности точек
-  Вычисление признаков треков (скорость, направление, длина пути)
-  Re-identification: идентификация одного автомобиля по косвенным признакам
-  Воспроизводимые результаты
-  Чистая архитектура с разделением ответственности

## Структура проекта

```
HardAutoLife/

 config.py              # Конфигурация ReID

```

## Формат входных данных

В SCHEMA.md описание формата JSON.

Пример:

```json
{
  "batch_id": "batch_001",
  "tracks": [
    {
      "track_id": "track_001",
      "source_id": "camera_01",
      "start_ts": "2024-01-15T10:00:00Z",
      "end_ts": "2024-01-15T10:00:05Z",
      "points": [
        {
          "ts": "2024-01-15T10:00:00Z",
          "lat": 55.7558,
          "lon": 37.6173
        }
      ]
    }
  ]
}
```

## Алгоритм Re-identification

Алгоритм использует двухэтапный подход:

1. **Жёсткие фильтры**: проверка совместимости по цвету, марке и источнику
2. **Сходство траекторий**: вычисление сходства по признакам движения
   - Средняя и медианная скорость
   - Направление движения
   - Длина пути
   - Географическая близость
   - Временной разрыв/перекрытие

Итоговый confidence вычисляется как взвешенная сумма признаков с применением сигмоиды для нормализации в диапазон [0, 1].

Треки объединяются в сущности (entities) если confidence >= порога (по умолчанию 0.75).

## Конфигурация

Параметры алгоритма ReID можно настроить в `config.py`:

- Пороги для фильтров (color, make)
- Веса признаков траектории
- Порог confidence для объединения
